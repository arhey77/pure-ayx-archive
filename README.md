# pure-ayx-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容来源多样，不针对任何特定域名或网站。

## 项目简介

`pure-ayx-archive` 是一个静态 HTML 页面归档仓库。这里存放的是各种独立页面的备份或快照，旨在以原始 HTML 格式保存内容，方便查阅和索引。

仓库不包含动态后端或数据库，所有页面均为纯前端静态文件。

## 目录说明

```
.
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 页面引用的静态资源（如样式、图片、脚本）
├── index.html      # 仓库首页 / 页面索引（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立的归档页面，命名方式为 `page-xxx.html` 或 `xxx/index.html`。
- `assets/`：按需存放页面所需的 CSS、JavaScript、图片等资源文件，目录结构尽量与页面保持对应。

## 页面归档说明

归档的页面遵循以下原则：

- 每个页面均为独立的 HTML 文件，可离线打开浏览。
- 页面内容保持原样，不进行修改或删减（除必要的路径调整外）。
- 归档时间、原始来源等信息可能记录在页面注释或附属说明文件中。
- 不保证所有页面内容的时效性、准确性或完整性。

## 维护说明

- 本仓库由维护者不定期更新，新增的归档页面将提交至 `main` 分支。
- 欢迎通过 Issue 提出归档建议，但不对所有请求做出承诺。
- 如需引用或分发仓库中的页面，请自行确认原始内容的授权情况。
- 仓库本身不承担页面内容可能带来的任何法律责任或争议。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的 HTML 文件。
3. 也可通过 GitHub Pages 等静态托管服务部署后在线访问。
